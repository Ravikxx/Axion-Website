What changed

Why we moved

Lumen started on a shared Hugging Face Space — free, easy to deploy, and good enough to get the model into people's hands quickly. But shared hosting has hard limits that started biting us.

The problem with shared hosting

Hugging Face Spaces run on shared infrastructure. When demand spikes — someone else's model goes viral, or multiple apps share the same GPU allocation — Lumen's response times climb and requests get throttled. We don't control the hardware, the scheduling, or the network.

For an AI lab building its own models, that's a ceiling we couldn't work around. We needed infrastructure we fully control: dedicated GPUs, no noisy neighbors, and the ability to scale on our terms.

What this means for you

Faster responses. Lumen now runs on dedicated hardware with no contention from other workloads. Prompts that took around 220 seconds on the shared Space now complete in 2–3 seconds.

More reliable. No more intermittent slowdowns when HF infrastructure is under load. We own the uptime.

Same endpoint. The API hasn't changed. api.amplifiedsmp.org still serves the same OpenAI-compatible /v1/chat/completions endpoint. No SDK changes, no config changes.

The model hasn't changed. This is the same Lumen 1.2.5 — same weights, same safety alignment, same benchmarks. The upgrade is purely infrastructure. Think of it as moving from a rented apartment to a house we built.
How we got here

The migration timeline

Moving off shared hosting while keeping the model available to users wasn't something we could do overnight. Here's how it unfolded.

✓

Lumen is fully live on dedicated infrastructure. Faster, more reliable, and under our complete control. Veil is also being migrated and will be available again shortly.
What's next

This migration is step one. Dedicated servers give us the foundation to run more experiments, serve more users, and iterate faster on Lumen 1.3. With full control over our infrastructure, we can now benchmark under realistic conditions, test scaling strategies, and push the model harder without worrying about hitting someone else's limits.
